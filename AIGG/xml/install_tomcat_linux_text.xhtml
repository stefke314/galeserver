1. Open the terminal.
  2. Create gale directory
    mkdir gale
  3. Go into the GALE folder
    cd gale
  4. Download the Tomcat 6 core installation files
    The tar.gz file with the Tomcat 6 core installation files can be downloaded from tomcat.apache.org. You can do this by hand with the browser but also with a commandline command:
    wget http://apache.mirror1.spango.com/tomcat/tomcat-7/v7.0.56/bin/apache-tomcat-7.0.56.tar.gz
  5. Extract the tar.gz file
    To extract the apache-tomcat-7.0.45.tar.gz file execute the following command:
    tar xf apache-tomcat-7.0.45.tar.gz
  6. Renaming the folder
    For convienience the folder apache-tomcat-7.0.56 will be renamed to 'tomcat':
    mv apache-tomcat-7.0.56 tomcat
  7. Start the Tomcat server for the first time
    To start the Tomcat server execute the following script:
    ./tomcat/bin/startup.sh
  8. Check if the server has started correctly
    Checking if the server has started correctly can be done by opening your browser and go to the localhost:8080 page. Now you should get the Tomcat page which is similar to:
    Another way to check if the server has started correctly is by looking at the Catalina logs. This log contains the errors, warnings and info messages that each servlet generates. If something went wrong it should be logged in the catalina.out file. To show the last lines of the Catalina log type the following command:
    tail ./tomcat/logs/catalina.out
    When the server has started successfully it should not show any errors and the message: "INFO: Server startup in xxxx ms" should be printed in the end.
    When you want to watch the log continuously you can also add the '-f' flag after the tail command. Then whenever there is written to the log file, this is shown in the terminal. You could of course also use any other text editor to browse through the complete logs.
  9. Shutting down the Tomcat server
    To shutdown the Tomcat server execute the following script:
    ./tomcat/bin/shutdown.sh
